HANINDO GROUP SITE — PHOTOS NEEDED
==================================================
Every slot below is already wired into a page. Drop a file with the
EXACT filename into the folder named in its section and it appears
automatically — no code changes needed. Until then the page shows a
styled "Photo needed" placeholder, so nothing looks broken.

Format: JPG for photos, PNG for logos. At least ~1000px wide for
heroes and ~800px for cards. Landscape suits heroes and cards.


FOLDER LAYOUT
--------------------------------------------------
/photos/ is grouped by company site, then by what the photo is for.
Put a new photo in the folder for the site that uses it:

  photos/brand/               company logos and the group lockup
  photos/shared/hero/         heroes used by more than one site
  photos/shared/certificates/ ISO and SKK certificates
  photos/partners/            principal / supplier logos (26)
  photos/customers/           customer logos, shared by both walls (39)

  photos/group/hero/          homepage and group-page heroes
  photos/group/divisions/     the division cards on the homepage
  photos/group/overseas/      Hanindo Overseas page

  photos/automotive/hero/     Hanindo Automotive banners
  photos/automotive/brands/   ATOM and WELDANN brand shots
  photos/automotive/products/ the 36 catalogue product shots
  photos/oil-and-gas/hero/    + /products/
  photos/printer-pos/hero/    + /products/
  photos/automation/hero/     + /products/
  photos/flowtech/hero/       + /sectors/

  photos/not-used/            on disk but referenced by no page

Folders with nothing in them yet hold a .gitkeep note saying which
filenames they are waiting for.

NOTE: the /fire-fighting/ section has its OWN photos folder
(/fire-fighting/photos/) which is already fully populated and is NOT
part of the layout above. Do not move those files and do not add
photos/fire-fighting/ — those pages reference their local folder as
plain "photos/...", so a root-level folder of the same name would
silently shadow nothing and just confuse the next person.

NOTES ON SOURCES
Most of these came from proxied search-result thumbnails, which are
small and, where the original had transparency, arrive with a grey
checkerboard flattened into the pixels. That grey was mapped back to
white on the affected files, and every logo was trimmed to its artwork
so the fixed-height cell scales them all to the same optical size.

The weakest files are adaro (200px) and mrt-jakarta (320px). They are
fine at 52px but soft if ever shown larger. Replace any of these with
press-kit or vector originals when they can be obtained — keep the same
filename and the page picks them up with no code change.

jict.png is the emblem only, with no wordmark, which matches the
company's own mark but reads less clearly at cell size than the others.

the rest of the wall. Fine at 52px tall, soft if ever shown larger.

airasia.png is white on a solid red block, so its cell reads as a red
rectangle where every other cell is white artwork on white. Deliberate —
it is a standard AirAsia lockup — but a red-on-white version would sit
more quietly if one turns up.

than its neighbours in the fixed-height cell. burger-king.png is nearly
square, so it sits the same way.

file supplied had the chevron sliced and was cut back to the wordmark
alone; the replacement is complete. It arrived white on IDEMIA purple,
which would have shown as a purple block in a wall of white cells, so it
was inverted to purple on white — the company's primary lockup, and the
same artwork, not a recolour of their brand.

SITA and Philippine Airlines are SVG, not PNG. Both raster versions were
cropped — SITA's final A and PAL's sunburst were sliced in the supplied
files — so they were replaced with public-domain vectors from Wikimedia
Commons. Being vector they cannot go soft or clip at any size, which
also settles the blur on the PAL logo. The partners wall already uses
SVG (opw.svg, piusi.svg), so nothing special is needed to serve them.
